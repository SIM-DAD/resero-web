NAME

resero — the free decrypter for age files.

SYNOPSIS
$ resero [--passphrase | --keyfile FILE | --simdad PRODUCT] FILE
$ resero --watch DIRECTORY           # folder watch mode
DESCRIPTION

Drag the file in. Pick an identity. Get your plaintext back. Resero is a small GUI for the age encryption format, free and MIT-licensed, from the same shop as Ibis, Lector, and TASS.

The age file format is small, modern, and well-specified, but the official tooling is command-line only. That is fine for sysadmins. It is less fine for a journalist on a fresh laptop, a researcher with an old archive, or a SIM DAD customer who wants their library back without reinstalling the app that wrote it. Resero is the missing GUI: open the file, hand it the right identity, get plaintext.

IDENTITY MODES
--passphrase
Decrypt with an scrypt-derived passphrase. Resero prompts you in a focused, single-purpose window. The passphrase never leaves the process and is zeroed on exit.
--keyfile FILE
Decrypt with an X25519 secret key supplied as a plain-text key file. Useful for files encrypted on another machine where the secret was exported once and is now portable.
--simdad PRODUCT
Auto-detect the secret from the OS keychain entry written by Ibis, Lector, or TASS. The right mode is usually obvious from the filename you dragged in.
--watch DIRECTORY
Folder-watch mode. Resero monitors the directory and decrypts new ciphertext as it arrives. Designed for scripted backup recovery and bulk archive workflows.
EXAMPLES
# Decrypt a single Lector library file using the OS keychain
$ resero --simdad lector  ~/Documents/draft.md.age
# Decrypt every .age file in a folder with a passphrase
$ resero --passphrase  ~/backups/
# Recover an Ibis archive on a fresh machine using an exported key
$ resero --keyfile ibis.age-key  ~/recovered/audio/
PORTABILITY

age files are byte-identical across Windows, macOS, and Linux. Only the keystore differs. Export your X25519 secret from the OS where you encrypted, import it on a fresh machine, decrypt the same files. No reinstall of the original product is required, and the recovery path is documented per-platform.

WHY THIS EXISTS

Every SIM DAD product encrypts at rest. We could ask you to take that on faith. Instead we ship the recovery tool ourselves and license it MIT. Resero exists so the encryption story is not "trust us." It is "here is the source, here is the binary, here is the file format spec it conforms to. Decrypt your own data without us in the loop."

No-lock-in is a claim. Resero is the proof.
